# Deployment Notes — Streamlit

> ⚠️ This file documents future deployment options. No deployment has been done yet.
> The app currently runs locally only. Do not push or deploy until local verification is complete.

---

## Current state

The app runs entirely locally and depends on:

1. **Precomputed parquet files** under `data/silver/` and `data/gold/` — generated by the pipeline scripts from raw Waymo TFRecords
2. **Scenario GIF previews** under `data/previews/` — generated by `generate_preview_gifs.py`
3. **Static diagram PNGs** under `data/diagrams/` — committed to the repo

None of items 1 or 2 are committed to the repo. They are generated locally and gitignored.

---

## What Streamlit Cloud would need

Streamlit Cloud deploys directly from a GitHub repo. It cannot run the full local pipeline (no access to raw Waymo TFRecords). The app therefore cannot be deployed as-is without a data strategy.

**Files required at deploy time:**

| File | In repo? | Strategy needed |
|---|---|---|
| `scripts/app.py` | ✅ Yes | Nothing needed |
| `requirements.txt` | ✅ Yes | Nothing needed |
| `proto/` compiled classes | ✅ Yes | Nothing needed |
| `data/diagrams/*.png` | ✅ Yes | Nothing needed |
| `data/silver/*.parquet` | ❌ No | See options below |
| `data/gold/*.parquet` | ❌ No | See options below |
| `data/previews/*.gif` | ❌ No | See options below |

---

## Future options for data artifacts

**Option A — Commit curated small artifacts**
- For a fixed 250-scenario public demo, parquet files are small (~13 KB – 200 KB each)
- GIFs are larger (~100–500 KB each × 250 scenarios)
- Could commit parquet files; host GIFs externally or use a subset
- Simplest approach for a stable public demo

**Option B — External artifact hosting**
- Store parquets and GIFs in an S3 bucket, GitHub Release, or similar
- App downloads them at startup
- Requires changes to `app.py` to fetch from remote URLs

**Option C — Rebuild locally, then deploy prebuilt**
- Run the full pipeline locally
- Commit the outputs temporarily to a deploy branch
- Deploy from that branch (not from main)
- Keep main clean of generated artifacts

**No option has been chosen or implemented yet.**

---

## Secrets and environment variables

The current app has no API keys or secrets. If remote artifact hosting is added, credentials would need to be set as Streamlit secrets.

---

## Deployment constraints

- Raw Waymo TFRecord files must **never** be uploaded anywhere (license terms)
- Parquet files contain only derived metrics — no raw sensor data — so they are safe to share
- GIFs are visualizations of trajectory coordinates — no raw sensor data — generally safe to share
- Confirm Waymo dataset terms before publishing any derived artifacts
